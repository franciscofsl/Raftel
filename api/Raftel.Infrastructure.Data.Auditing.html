Namespace Raftel.Infrastructure.Data.Auditing

Classes

AuditOptions

Options that determine which entity types are tracked by the entity-change audit system. Entities are opt-in: only types explicitly registered via Audit<TEntity>() are audited.

EntityChangeSnapshot

EF Core-agnostic representation of a single entity change, produced by IChangeSnapshotExtractor and consumed by IAuditLogFactory.

PropertyChangeSnapshot

EF Core-agnostic representation of a single property value change, produced by IChangeSnapshotExtractor and consumed by IAuditLogFactory.

Interfaces

IAuditLogFactory

Builds an AuditLog aggregate from EF Core-agnostic change snapshots. Has no dependency on EF Core or the dependency injection container, making it fully unit-testable in isolation.

IAuditStore

Persists an AuditLog aggregate as part of an ongoing EF Core save operation.

IAuditValueSerializer

Converts an arbitrary property value (primitive, value object, or nested value object) into a human-readable string suitable for audit storage. Has no dependency on EF Core.

IChangeSnapshotExtractor

Reads the EF Core change tracker and produces EF Core-agnostic snapshots for every audited entity that was added, modified or deleted. This is the only audit component directly coupled to EF Core.
